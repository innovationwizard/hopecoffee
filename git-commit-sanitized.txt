fix(january-2026): reconcile app to SSOT cell-for-cell + fix 3 latent calc bugs

The "Contexto del Mes" card was showing a pooled margin of 18.29% that
matched neither Block 1 (15.594%) nor Block 2 (13.893%). Root cause was
compound: the DB was imported once before Friday's SSOT fix and had been
carrying stale MateriaPrima values ever since, AND calculateContract had
three latent bugs that only surfaced when I exercised it directly during
reconciliation. Fixed the data, fixed the calc engine, added a regression
gate, and documented the whole thing.

All 4 January contracts (P30172, P40028, P40022, P40129) across 2 shipments
(Bloque 1 / Exportadora, Bloque 2 / Finca Danilandia) now match Enero.xlsx
cell-for-cell in Supabase prod.

CALC ENGINE FIXES (src/lib/services/calculations.ts):

1. Collapsed billing to a single kg path. Deleted the LIBRAS_ESPANOLAS
   branch, which was dead buggy code that would have double-multiplied
   the kg uplift if anyone had ever flipped the enum. Per business rules
   1.5/1.6/2.3 there is exactly one billing formula. tipoFacturacion
   is now @deprecated on the input interface and ignored.

2. Replaced the x 1.01411 approximation with the canonical SSOT formula
   sacos69 x 69 x 2.2046 x (precioBolsaDif / 100). The 1.01411 constant
   is a 5-decimal truncation of (69 x 2.2046 / 100) / 1.5 = 1.01411733...
   and drifts ~$1/contract on realistic volumes. Now matches the SSOT
   col O cell-for-cell.

3. Fixed a latent unit bug: gastosExportacion was gastosPerSaco x sacos69,
   but business rules 1.7 is unambiguous - "rate per quintal x total
   quintales", and a quintal is 46 kg not 69 kg. Fixed to x sacos46. Did
   not fire in prod because Contract.gastosExport was being populated via
   the shipment-level exportCostConfig join, bypassing the formula. Fired
   immediately when I ran calculateContract directly in the reconciliation.

4. Added facturacionKgsOverride support. When the new Contract field is
   non-null, both facturacionLbs and facturacionKgs take the override
   verbatim, preserving downstream consistency.
